.. py:method:: add_pin_pair_rlc(first_pin: str, second_pin: str, resistance: str | float | int | None = None, inductance: str | float | int | None = None, capacitance: str | float | int | None = None, is_parallel: bool = False, resistance_enabled: bool = False, inductance_enabled: bool = False, capacitance_enabled: bool = False)

   
   Append a pin-pair RLC model between two component pins.


   :Parameters:

       **first_pin** : :class:`python:str`
           Name of the first pin, e.g. ``"1"``.

       **second_pin** : :class:`python:str`
           Name of the second pin, e.g. ``"2"``.

       **resistance** : :class:`python:str`, :class:`python:float`, :obj:`or` :data:`python:None`, :obj:`optional`
           Resistance value, e.g. ``"100ohm"`` or ``100.0``.

       **inductance** : :class:`python:str`, :class:`python:float`, :obj:`or` :data:`python:None`, :obj:`optional`
           Inductance value, e.g. ``"1nH"``.

       **capacitance** : :class:`python:str`, :class:`python:float`, :obj:`or` :data:`python:None`, :obj:`optional`
           Capacitance value, e.g. ``"100nF"``.

       **is_parallel** : :ref:`bool <python:bltin-boolean-values>`, :obj:`optional`
           ``True`` for a parallel RLC topology.  Default is ``False``
           (series).

       **resistance_enabled** : :ref:`bool <python:bltin-boolean-values>`, :obj:`optional`
           Activate the resistance element.  Default is ``False``.

       **inductance_enabled** : :ref:`bool <python:bltin-boolean-values>`, :obj:`optional`
           Activate the inductance element.  Default is ``False``.

       **capacitance_enabled** : :ref:`bool <python:bltin-boolean-values>`, :obj:`optional`
           Activate the capacitance element.  Default is ``False``.













   .. rubric:: Examples

   r1 = cfg.components.add("R1", part_type="resistor")
   r1.add_pin_pair_rlc("1", "2", resistance="100ohm", resistance_enabled=True)

.. py:method:: set_solder_ball_properties(shape: str = 'cylinder', diameter: str | None = None, height: str | None = None, material: str = 'solder', mid_diameter=None, orientation: str = 'chip_down', reference_designator: str | None = None)

   
   Configure solder-ball geometry for this component.


   :Parameters:

       **shape** : :class:`python:str`, :obj:`optional`
           Solder-ball shape.  ``"cylinder"`` (default), ``"spheroid"``, or
           ``"no_solder_ball"``.

       **diameter** : :class:`python:str`, :obj:`optional`
           Outer diameter, e.g. ``"150um"``.  When *None* and a live EDB
           session is attached the smallest pin pad size found on the
           component is used automatically.  Falls back to ``"150um"`` if
           the pad size cannot be determined.

       **height** : :class:`python:str`, :obj:`optional`
           Solder-ball height, e.g. ``"100um"``.  When *None* the height is
           set to ``2 * diameter / 3``.

       **material** : :class:`python:str`, :obj:`optional`
           Material name.  Default is ``"solder"``.

       **mid_diameter** : :class:`python:str` or :data:`python:None`, :obj:`optional`
           Mid-diameter for spheroid shape.  Defaults to *diameter* when
           *None*.

       **orientation** : :class:`python:str`, :obj:`optional`
           Die orientation for IC components.  ``"chip_down"`` (default) or
           ``"chip_up"``.

       **reference_designator** : :class:`python:str`, :obj:`optional`
           Override the component reference designator used when querying pin
           sizes from EDB.  When *None* ``self.reference_designator`` is
           used.













   .. rubric:: Examples

   u1.set_solder_ball_properties("cylinder", "150um", "100um")
   u1.set_solder_ball_properties()  # auto-sizes from pin pads
